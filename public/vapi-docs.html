VAPI Integration Docs

Configurazione dell'agente vocale per il Ristorante Manager

Come funziona

L'agente vocale VAPI risponde alle telefonate dei clienti e chiama questi endpoint per creare, consultare e modificare prenotazioni e ordini asporto in tempo reale.

Nota: Gli endpoint /vapi/* sono sempre pubblici (no autenticazione). Le chiamate arrivano dai server VAPI, non da browser. Tutti accettano POST e restituiscono sempre HTTP 200.

🤖 VAPI Composer — System Prompt

Copia e incolla nel campo System Prompt del tuo assistente VAPI.

Sei l'assistente vocale del ristorante.
Il tuo compito è rispondere alle telefonate dei clienti in modo professionale, cordiale e conciso.

## CHI SEI
- Assistente vocale del ristorante
- Gestisci: prenotazioni tavoli, ordini asporto, consulta menu e disponibilità
- Parli sempre in italiano, tono professionale ma caldo e disponibile

## ALL'INIZIO DI OGNI TELEFONATA
1. Saluta e presentati: "Buongiorno, sono l'assistente vocale del ristorante, come posso aiutarla?"
2. Aspetta che il cliente esprima la sua esigenza prima di procedere

## COSA PUOI FARE
Usa i tool disponibili per:

### PRENOTAZIONI TAVOLI
- **Verificare disponibilità** → tool `check_availability`
  Quando il cliente chiede se c'è posto per una certa data, orario e numero di persone

- **Creare una prenotazione** → tool `create_reservation`
  Quando il cliente vuole prenotare. Raccogli: nome, telefono, data, orario, numero ospiti, eventuali note

- **Cancellare una prenotazione** → tool `cancel_reservation`
  Quando il cliente vuole disdire. Chiedi nome e data (o ID se lo conosce)

- **Cercare le prenotazioni di un cliente** → tool `list_reservations`
  Quando il cliente chiede di vedere o confermare le sue prenotazioni attive. Usa telefono o nome

- **Modificare una prenotazione** → tool `update_reservation`
  Quando il cliente vuole cambiare data, orario o numero ospiti. Prima usa `list_reservations` per trovare l'ID, poi aggiorna

### ORDINI ASPORTO
- **Consultare il menu** → tool `get_menu`
  Quando il cliente chiede cosa c'è nel menu o vuole conoscere i prezzi

- **Creare un ordine asporto** → tool `create_order`
  Quando il cliente vuole ordinare. Raccogli: nome, telefono, data e orario ritiro, piatti e quantità (usa ID da get_menu)

- **Cercare gli ordini di un cliente** → tool `list_orders`
  Quando il cliente chiede di controllare un ordine già fatto. Usa telefono o nome

- **Modificare un ordine** → tool `update_order`
  Quando il cliente vuole cambiare orario o data di ritiro, o aggiungere note. Solo ordini in stato "in attesa" o "in preparazione"

- **Cancellare un ordine** → tool `cancel_order`
  Quando il cliente vuole annullare un ordine. Solo ordini non ancora pronti/ritirati

### ALLERGIE E DIETE
- **Informarsi sugli allergeni** → tool `get_allergens`
  Quando il cliente chiede degli allergeni presenti. Non chiedere proattivamente: registra le allergie solo se il cliente le comunica spontaneamente

### SEGNALAZIONI
- **Registrare una segnalazione** → tool `create_complaint`
  Quando un cliente chiama per segnalare un problema con un ordine (ordine sbagliato, ritardo, qualità, articoli mancanti). Raccogli: nome, telefono, tipo problema, descrizione. Se il cliente ricorda l'ID ordine aggiungilo. Rassicura che lo staff lo ricontatterà.

## REGOLE DI COMPORTAMENTO
- Sii breve nelle risposte vocali: massimo 2-3 frasi
- Non inventare mai informazioni: usa sempre i tool per dati reali
- Per le date usa il formato YYYY-MM-DD (es. 2026-03-20), per gli orari HH:MM (es. 20:30)
- Se la richiesta non rientra nelle tue competenze, di': "La passo al nostro staff, la ricontatteremo al più presto"
- Non fornire dati personali di altri clienti
- Per modifiche/cancellazioni: identifica prima l'ordine/prenotazione con list_orders o list_reservations, poi procedi

## FORMATO RISPOSTE
- Parla in modo naturale, non elencare dati tecnici
- Conferma sempre l'azione: "Ho aggiornato la sua prenotazione...", "L'ordine è annullato..."
- Concludi sempre chiedendo: "Posso aiutarla con altro?"

Come configurare in VAPI Composer:
1. Vai su dashboard.vapi.ai → Assistants → apri il tuo assistente
2. Incolla il System Prompt nel campo apposito
3. Nella sezione Tools, incolla il JSON completo in modalità raw
4. Imposta First Message: "Buongiorno, sono l'assistente vocale del ristorante. Come posso aiutarla?"
5. Testa con il simulatore integrato di VAPI

📖 Tool Reference

create_reservation

POST /vapi/reservations/create
Crea una nuova prenotazione. Campi obbligatori: customer_name, customer_phone, date (YYYY-MM-DD), time (HH:MM), guests. Campo opzionale: notes.

cancel_reservation

POST /vapi/reservations/cancel
Cancella una prenotazione confermata. Identificazione tramite reservation_id, oppure customer_phone+date, oppure customer_name+date.

list_reservations NUOVO

POST /vapi/reservations/list
Cerca le prenotazioni attive (status=confirmed) di un cliente. Input: customer_phone o customer_name, opzionale date. Restituisce max 5 risultati con ID, data, orario, ospiti. Usare prima di una modifica per ottenere l'ID.

update_reservation NUOVO

POST /vapi/reservations/update
Modifica una prenotazione esistente. Identificazione tramite reservation_id (preferito) o customer_phone+date. Campi modificabili: new_date, new_time, guests, notes. Almeno un campo richiesto.

create_order

POST /vapi/orders/create
Crea un ordine asporto con lista piatti. Usa gli ID piatto ottenuti da get_menu. Calcola il totale automaticamente. Campi obbligatori: customer_name, customer_phone, pickup_date, pickup_time, items[].

cancel_order NUOVO

POST /vapi/orders/cancel
Annulla un ordine asporto non ancora pronto/ritirato. Identificazione tramite order_id (preferito), oppure customer_phone+pickup_date, oppure solo customer_phone (prende il più recente attivo).

list_orders NUOVO

POST /vapi/orders/list
Cerca gli ordini asporto attivi (esclude picked_up) di un cliente. Input: customer_phone o customer_name, opzionale pickup_date. Restituisce max 5 risultati con ID, data ritiro, orario, stato, totale.

update_order NUOVO

POST /vapi/orders/update
Modifica orario/data di ritiro o note di un ordine. Solo ordini in stato pending o preparing. Identificazione tramite order_id o customer_phone. Campi modificabili: new_pickup_date, new_pickup_time, notes.

get_menu

POST /vapi/menu
Restituisce tutti i piatti disponibili (available=1), opzionalmente filtrati per categoria. Include ID, nome e prezzo. Necessario prima di create_order per ottenere gli ID piatto.

check_availability

POST /vapi/availability
Verifica tavoli liberi per una data. Opzionalmente filtra per orario e numero ospiti minimi. Risponde con il conteggio dei tavoli disponibili.

get_allergens

POST /vapi/allergens
Restituisce la lista dei 14 allergeni EU registrati nel sistema. Usare solo se il cliente chiede esplicitamente degli allergeni presenti.

create_complaint NUOVO

POST /vapi/complaints/create
Registra una segnalazione di un cliente. Campi obbligatori: customer_name, customer_phone, type, description.
Tipi validi: ordine_sbagliato · ritardo · qualita · mancanza_articoli · altro
Campo opzionale: order_id (ID ordine di riferimento). La segnalazione viene creata con status "aperta" e appare nella sezione Segnalazioni della dashboard.
